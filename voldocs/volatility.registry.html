v.registry : module documentation

Part of volatility

This module implements a class registry.

We scan the memory_plugins directory for all python files and add those classes which should be registered into their own lookup tables. These are then ordered as required. The rest of Volatility will then call onto the registered classes when needed.

This mechanism allows us to reorganise the code according to functionality. For example we may include a Scanner, Report and File classes in the same plugin and have them all automatically loaded.
