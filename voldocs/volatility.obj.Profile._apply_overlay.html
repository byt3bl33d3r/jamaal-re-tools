v.o.P._apply_overlay(self, type_member, overlay) : method documentation

Part of volatility.obj.Profile

Update the overlay with the missing information from type.

Basically if overlay has None in any slot it gets applied from vtype.

We make extensive use of copy.deepcopy to ensure we don't modify the original variables. Some of the calls may not be necessary (specifically the return of type_member and overlay) but this saves us the concern that things will get changed later and have a difficult-to-track down knock-on effect.
